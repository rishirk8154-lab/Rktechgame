JUnit Assertions kya hain?

Assertions JUnit ke test methods me wo checks hote hain jo verify karte hain ki expected aur actual results milte hain ya nahi. Agar assertion fail hota hai to test fail ho jata hai aur bug pakad me aata hai.

Common Assertions Types

  • assertEquals(expected, actual) - Verify karta hai ki do values barabar hain.
  • assertTrue(condition) - Verify karta hai ki condition true hai.
  • assertFalse(condition) - Verify karta hai ki condition false hai.
  • assertNull(object) - Verify karta hai ki object null hai.
  • assertNotNull(object) - Verify karta hai ki object null nahi hai.
  • assertArrayEquals(expectedArray, actualArray) - Arrays ki equality check karta hai.
  • assertThrows(exceptionClass, executable) - Verify karta hai ki code block expected exception throw karta hai.

JUnit Assertions Example

import static org.junit.jupiter.api.Assertions.*;
import org.junit.jupiter.api.Test;

public class AssertionsDemo {

    @Test
    public void testAssertions() {
        assertEquals(10, 5 + 5);
        assertTrue(3 < 4);
        assertFalse(5 > 6);
        assertNull(null);
        assertNotNull("JUnit");
        assertArrayEquals(new int[]{1, 2, 3}, new int[]{1, 2, 3});
        assertThrows(ArithmeticException.class, () -> {
            int x = 1 / 0;
        });
    }
}

JUnit Test Suites kya hai?

Test suites me multiple test classes ko ek saath group karke ek run me chalaya jata hai. Isse bade projects ke tests organized aur manageable ho jate hain.

JUnit Test Suite Example (JUnit 4)

import org.junit.runner.RunWith;
import org.junit.runners.Suite;

@RunWith(Suite.class)
@Suite.SuiteClasses({
    TestClass1.class,
    TestClass2.class,
    TestClass3.class
})
public class AllTestsSuite {
    // Yeh class khud me koi code nahi rakhti, sirf annotations se test classes ko group karti hai
}

Is example me TestClass1, TestClass2 aur TestClass3 ko ek suite me group kiya gaya hai.
