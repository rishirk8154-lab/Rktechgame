Writing Simple Unit Tests

Introduction

Unit tests are small pieces of code written to verify that a specific function or method works correctly. Writing simple unit tests helps catch bugs early and makes code more maintainable.

Why Write Unit Tests?

  • Ensure individual functions work as intended.
  • Catch errors early in development.
  • Provide documentation of expected behavior.
  • Support safe refactoring and future enhancements.

Steps to Write Simple Unit Tests

  1. Identify the Unit: Choose the smallest testable part of your code (e.g., a function or method).
  2. Write Test Function: Create a test function/method that calls the unit with test inputs.
  3. Assert Expected Results: Use assertions to verify output matches the expected result.
  4. Run Tests: Execute tests with a testing framework to see if they pass or fail.
  5. Debug & Refine: Fix bugs found and refine tests for better coverage.

Example: Simple Calculator Unit Test (Python)

class Calculator:
    def add(self, a, b):
        return a + b

    def subtract(self, a, b):
        return a - b


import unittest

class TestCalculator(unittest.TestCase):
    def setUp(self):
        self.calc = Calculator()

    def test_add(self):
        self.assertEqual(self.calc.add(3, 4), 7)
        self.assertEqual(self.calc.add(-1, 1), 0)

    def test_subtract(self):
        self.assertEqual(self.calc.subtract(10, 4), 6)
        self.assertEqual(self.calc.subtract(0, 5), -5)

if __name__ == '__main__':
    unittest.main()

Tips for Effective Unit Tests

  • Keep tests simple and focused on one behavior.
  • Use clear and descriptive names for tests.
  • Test positive, negative, and boundary cases.
  • Run tests frequently to catch regressions early.
  • Mock external dependencies to isolate tested units.

Conclusion

Writing simple unit tests is a foundational skill for developers. It improves code reliability and helps maintain a bug-free codebase by ensuring each part works correctly.

Parichay

Unit tests chhote code ke tukde hote hain jo ye verify karte hain ki kisi specific function ya method kaam sahi karta hai. Simple unit tests likhne se bugs jaldi milte hain aur code maintain karna asaan hota hai.

Unit Tests Kyun Likhein?

  • Har function ka sahi kaam karna confirm karen.
  • Development ke shuruaat me hi error pakadna.
  • Kod ka expected behavior document karna.
  • Refactor karna aur naye features safely add karna.

Simple Unit Test Likhnay Ke Steps

  1. Unit Ko Pehchanein: Code ke sabse chhote test karne jogh hissa (jaise function ya method) chunein.
  2. Test Function Banayein: Aisa function likhein jo test input ke sath unit ko call kare.
  3. Expected Result Ko Assert Karein: Dekhein ki output expected result ke barabar hai ya nahi.
  4. Tests Chalayein: Testing framework se tests run karein aur pass/fail dekhein.
  5. Debug Karein: Agar bugs milen to fix karen aur tests ko behtar banayein.
